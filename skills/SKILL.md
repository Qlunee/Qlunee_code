# PDF Processing Skill (UV Optimized)

You now have expertise in PDF manipulation using modern Python package management.

## 🛠 Pre-requisites & Environment
Before executing Python code, check if the required libraries are installed. If not, use `uv add`.

```bash
# Check if a library is installed
uv pip list | grep pymupdf

# Install if missing
uv add pymupdf reportlab pdfkit
```

## 📖 Reading PDFs

**Option 1: Quick text extraction (CLI)**
```bash
# Faster if poppler-utils is installed
pdftotext input.pdf - 
```

**Option 2: Programmatic extraction (PyMuPDF)**
*Requires: `uv add pymupdf`*
```python
import fitz  # PyMuPDF
doc = fitz.open("input.pdf")
# Extract all text
full_text = "".join([page.get_text() for page in doc])
print(full_text)
```

## ✍️ Creating PDFs

**Option 1: From Markdown (via Pandoc)**
```bash
# System dependency: pandoc
pandoc report.md -o report.pdf
```

**Option 2: Pure Python (ReportLab)**
*Requires: `uv add reportlab`*
```python
from reportlab.pdfgen import canvas
from reportlab.lib.pagesizes import letter

c = canvas.Canvas("output.pdf", pagesize=letter)
c.drawString(100, 750, "Hello, this PDF was generated by an Agent!")
c.save()
```

## 🔄 Merging & Splitting

*Requires: `uv add pymupdf`*

```python
import fitz

def merge_pdfs(input_paths, output_path):
    result = fitz.open()
    for path in input_paths:
        with fitz.open(path) as m:
            result.insert_pdf(m)
    result.save(output_path)
```

## 💡 Best Practices for Agent

1. **Self-Correction**: If a `ModuleNotFoundError` occurs, immediately run `uv add <package_name>` and retry the task.
2. **Persistence**: Using `uv add` ensures that `pyproject.toml` is updated, making the skill environment persistent for future subagents.
3. **Efficiency**: For very large PDFs, process page-by-page using `doc.pages()` iterator to save memory.
